To file GSTR-1 from Google Sheets, you have two main paths: the **Automated Path** (coding a custom script) or the **Manual Path** (using the government’s utility).

### 1. The Automated Path (Best for Developers)

You can use **Google Apps Script** to fetch your spreadsheet data and convert it into the required JSON format.

* **How it works:** You write a script that iterates through your "Sales" sheet, maps the columns to GST JSON keys (like `inum`, `idt`, `txval`), and triggers a file download.
* **The Workflow:**
1. **Map Data:** Ensure your Google Sheet columns exactly match the data required for B2B, B2CS, and HSN tables.
2. **Generate JSON:** The script converts the rows into a nested JSON object.
3. **Download & Upload:** The user clicks a button in the sheet, gets a `.json` file, and uploads it to the [GST Portal](https://www.gst.gov.in/) under **Returns Dashboard > GSTR-1 > Prepare Offline**.




---

### 2. The Manual Path (No Coding Required)

If you don't want to build a custom script, you can use the official **GST Offline Tool**.

* **The Workflow:**
1. **Download Template:** Download the official GSTR-1 Excel Macro template from the GST Portal.
2. **Copy-Paste:** Copy your sales data from Google Sheets and paste it into the respective tabs of the Excel template.
3. **Generate JSON:** Open the "GST Offline Tool" (desktop app), import this Excel file, and click **Generate JSON**.
4. **Upload:** Upload that JSON file to the portal.



---

### Comparison of Methods

| Feature | Automated (Google Apps Script) | Manual (Offline Utility) |
| --- | --- | --- |
| **Speed** | **Instant** (one click) | Slow (copy-pasting required) |
| **Error Risk** | Low (if script logic is correct) | High (manual errors during paste) |
| **Setup Effort** | High (requires writing code) | Low (just download the tool) |
| **Internet** | Required for Google Sheets | Offline (until upload) |

---

### 3. Which one should you choose?

* **Choose Automated** if you are building this for a client or have more than 50-100 invoices a month. It eliminates human error.
* **Choose Manual** if you are doing a one-time filing or have very few transactions.

### Critical Tip for 2026:

The GST Portal has implemented a **3-year hard stop**. You cannot file any GSTR-1 that is more than 3 years old. Additionally, ensure your Google Sheet includes the **HSN Summary (Table 12)**; the portal will reject any JSON that does not have HSN data for every invoice.

To ensure your JSON is 100% compliant for direct upload to the GST portal, your Google Sheet (the data source) must capture specific fields that the GSTN schema requires.

If you miss any of these "Master Columns," the portal will reject the entire JSON file during the validation phase.

### 1. The "B2B" Master Columns (Sales to GST Registered Businesses)

These are for Table 4A, 4B, 4C, 6B, 6C.

* **Recipient GSTIN:** (15-digit alphanumeric)
* **Receiver Name:** (Optional for JSON, but good for records)
* **Invoice Number:** (Max 16 characters, only `/` and `-` allowed)
* **Invoice Date:** (Format: `DD-MM-YYYY`)
* **Invoice Value:** (Total amount including tax)
* **Place of Supply:** (The 2-digit State Code, e.g., `27` for Maharashtra)
* **Reverse Charge:** (`Y` or `N`)
* **Invoice Type:** (`Regular`, `SEZ with payment`, `SEZ without payment`, etc.)
* **Tax Rate:** (0, 5, 12, 18, 28)
* **Taxable Value:** (Value before tax)
* **Integrated Tax (IGST):** (Calculated if POS $\neq$ Seller State)
* **Central Tax (CGST):** (Calculated if POS $=$ Seller State)
* **State Tax (SGST):** (Calculated if POS $=$ Seller State)
* **Cess:** (If applicable)

---

### 2. The "B2CS" Master Columns (Sales to Unregistered/Consumers)

These are for Table 7 (Small B2C).

* **Type:** (`OE` for Other than E-commerce, `E` for E-commerce)
* **Place of Supply:** (2-digit State Code)
* **Tax Rate:** (The slab applied)
* **Taxable Value:** (Total taxable value for that rate/state combination)
* **Cess:** (If applicable)

---

### 3. The "HSN Summary" Master Columns (Mandatory)

Table 12 is now the most common cause of JSON rejection. Your data recording **must** include:

* **HSN Code:** (minimum 4 or 6 digits depending on turnover)
* **Description:** (Brief description of goods/services)
* **UQC:** (Unit Quantity Code—must match GST portal list, e.g., `KGS`, `PCS`, `NOS`)
* **Total Quantity:** (Numeric)
* **Total Value:** (Total amount including tax)
* **Total Taxable Value:** (Amount before tax)
* **IGST/CGST/SGST Amount:** (Individual tax totals per HSN)

---

### 4. The "Document Issued" Master Columns

Table 13 tracks the sequence of your paperwork:

* **Nature of Document:** (e.g., "Invoices for outward supply")
* **From Serial No:** (First invoice of the month)
* **To Serial No:** (Last invoice of the month)
* **Total Number:** (Count of documents)
* **Cancelled:** (Count of deleted/cancelled invoices)

---

### 5. Essential Validation Tips for your Google Sheet

To make your JSON generation "bulletproof," apply these rules to your recording columns:

1. **State Codes:** Never record the name (e.g., "Delhi"). Use the code (`07`).
2. **Date Formatting:** Use a custom date format in Google Sheets to force `DD-MM-YYYY`. The portal will reject `YYYY-MM-DD` or `DD/MM/YY`.
3. **The "Math" Check:** Your JSON will fail if:
* *Taxable Value* $\times$ *Rate* $\neq$ *Total Tax*.
* *Taxable Value* $+$ *Total Tax* $\neq$ *Invoice Value*.


4. **No Nulls:** If a tax amount (like Cess) is zero, the JSON should usually contain `0`, not a blank string or `null`.

### Recommendation for your App Logic:

Since you are recording data in Google Sheets, you should have **one row per Tax Rate per Invoice**.
*Example:* If one invoice has 5% items and 18% items, you should record it as two rows in your sheet so your script can easily map them into the `itms` array in your JSON.

Using the correct **Unit Quantity Code (UQC)** is non-negotiable for GSTR-1. If you type "Kilo" instead of "KGS," the portal will reject your JSON file immediately.

Here is the standardized list of UQCs you should use as a dropdown menu in your Google Sheet to ensure your data recording is 100% compliant.

### 1. Most Common UQCs

These cover roughly 90% of general business transactions in India.

| UQC Code | Description | Best For |
| --- | --- | --- |
| **NOS** | NUMBERS | Electronics, furniture, discrete items |
| **PCS** | PIECES | Garments, spare parts, glassware |
| **KGS** | KILOGRAMS | Grains, chemicals, steel, weighted goods |
| **MTR** | METERS | Fabric, cables, wires |
| **BOX** | BOX | FMCG multipacks, kits |
| **SET** | SETS | Combined products (e.g., Sofa set) |
| **LTR** | LITRES | Oils, milk, liquids (Note: some systems use **KLR** for bulk) |
| **OTH** | OTHERS | Anything not listed above |

---

### 2. Full Standardized List (For Your App Logic)

If your app needs to handle specialized industries (Construction, Pharma, etc.), include these:

* **Weight:** `GMS` (Grams), `QTL` (Quintal), `TON` (Tonnes), `MTS` (Metric Ton).
* **Volume:** `MLT` (Millilitre), `CAN` (Cans), `BTL` (Bottles), `DRM` (Drums).
* **Area/Length:** `SQF` (Square Feet), `SQM` (Square Meters), `YDS` (Yards), `CMS` (Centimeters).
* **Packaging:** `BAG` (Bags), `PAC` (Packs), `CTN` (Cartons), `BUN` (Bunches), `BDL` (Bundles).
* **Pharma/Misc:** `TBS` (Tablets), `TUB` (Tubes), `DOZ` (Dozens), `THD` (Thousands).

---

### 3. Implementation Strategy for your Google Sheet

To make your JSON "Direct Upload" ready, I recommend setting up your "Sales" sheet with these rules:

1. **Data Validation:** In your `UQC` column, go to **Data > Data Validation** and restrict the entries to the codes above.
2. **Case Sensitivity:** Always store these as **UPPERCASE**. While some JSON parsers are flexible, the GSTN master schema prefers `KGS` over `kgs`.
3. **The "OTH" Rule:** If a user sells a service or a unique unit (like "Consulting Hours"), instruct them to use `OTH` (Others). The GST portal requires a UQC even for services; in those cases, the quantity is usually `1` and UQC is `OTH`.

### 4. Why this matters for your JSON

In your `hsn` array, the portal expects the data to look exactly like this:

```json
{
  "hsn_sc": "8471", 
  "uqc": "NOS",
  "qty": 10,
  "txval": 50000.00,
  ...
}

```

If `uqc` contains anything other than the 3-letter codes above, the upload status will show **"Error in JSON"** after the user uploads it.

To automate GSTR-1 filing from Google Sheets, your data recording must be highly structured. Think of your spreadsheet as a "Database" where each column satisfies a specific requirement of the GST JSON schema.

Below are the exact columns you should set up, grouped by the specific sections they will eventually populate in your JSON.

---

### 1. Master Recording Columns (Global)

These columns are required for almost every row to identify the transaction context.

| Column Name | Data Type | Validation Rule / Purpose |
| --- | --- | --- |
| **Transaction Type** | Dropdown | B2B, B2CS (Small), B2CL (Large), CDNR (Credit/Debit) |
| **Recipient GSTIN** | Text | 15-digit alphanumeric. **Mandatory for B2B.** |
| **Invoice Number** | Text | Max 16 chars. Only `-` and `/` special chars allowed. |
| **Invoice Date** | Date | Must be formatted as **`DD-MM-YYYY`**. |
| **Place of Supply** | Dropdown | **2-digit State Code only** (e.g., `27` for Maharashtra). |

---

### 2. Transaction Details (The "Money" Columns)

Each row in your sheet should represent a **unique Tax Rate** for an invoice. If an invoice has 12% and 18% items, use two rows with the same Invoice Number.

| Column Name | Data Type | Formula / Rule |
| --- | --- | --- |
| **Taxable Value** | Number | Value before GST. |
| **GST Rate** | Dropdown | Must be one of: `0`, `0.25`, `3`, `5`, `12`, `18`, `28`. |
| **IGST Amount** | Number | Calculated if POS $\neq$ Your State. |
| **CGST Amount** | Number | Calculated if POS $=$ Your State. |
| **SGST Amount** | Number | Calculated if POS $=$ Your State. |
| **Cess Amount** | Number | Enter `0` if not applicable. |
| **Total Invoice Value** | Number | Taxable Value + Total GST. |

---

### 3. HSN Summary Columns (Table 12 - Mandatory)

From May 2025 onwards, Table 12 is mandatory and must match the totals in your B2B/B2C sections.

| Column Name | Data Type | Rule |
| --- | --- | --- |
| **HSN/SAC Code** | Text | **4-digit** (Turnover < 5Cr) or **6-digit** (Turnover > 5Cr). |
| **UQC** | Dropdown | Use only official codes like **NOS, KGS, PCS, OTH**. |
| **Total Quantity** | Number | Sum of items for that HSN. |

---

### 4. Logic for your Google Apps Script

When you write your script to generate the JSON, it should perform these "Silent Checks" to prevent portal errors:

* **Zero-Padding:** Ensure the `pos` (Place of Supply) is always two digits (e.g., `7` becomes `"07"`).
* **Case Sensitivity:** Convert all GSTINs to **UPPERCASE** before putting them in the JSON.
* **Grouping:** The script should "Group" rows with the same Invoice Number into a single `inv` object in the JSON, but keep the `itms` (items) as an array within it.

---

### Pro-Tip: The "Document Summary" (Table 13)

The portal now requires a summary of document sequences. I recommend having a small **separate table** or a hidden sheet in your Google Sheet that tracks:

1. **Start Invoice No.** (e.g., `INV/26/001`)
2. **End Invoice No.** (e.g., `INV/26/050`)
3. **Total Count** (e.g., `50`)
4. **Cancelled Count** (e.g., `2`)
